## Operating System

next-ship is designed to work on macOS, Linux and Windows. While next-ship itself is platform-agnostic, the tooling and dependencies we use have different requirements.

We've tested and confirmed that next-ship works on the following combinations of operating systems, Node.js versions and next-ship versions:

| Operating system | next-ship version | Node.js version | Notes |
| ---------------- | ----------------- | --------------- | ----- |
| macOS Sequoia 15.0.1 (24A348) | 2.14.3 | 20.12.2 |  |
| Ubuntu 24.04 Arm64 | 2.14.3 | 20.18.0 |  |
| Fedora, Release 41 | 2.14.3 | 22.11.0 |  |
| Windows 11 Pro 24H2 (26100.2033) | 2.14.3 | 20.18.0 | Turborepo only supports specific architectures. `windows ia32` is not supported. |

We're aware of issues on [non-Ubuntu Linux distributions](https://github.com/vercel/next-ship/issues/238). While we don't officially support them, we'd love to know if you get it working!

## Package Manager

next-ship defaults to using [pnpm](https://pnpm.io/) as a package manager.

## Mintlify CLI

We use the [Mintlify CLI](https://mintlify.com/docs/development) to preview the [docs](/apps/docs) locally.

## Accounts

next-ship relies on various SaaS products. You will need to create accounts with the following services then set the API keys in your [environment variables](/docs/setup/env):

- [Clerk](https://clerk.com), for [authentication](/packages/authentication).
- [Google Analytics](https://developers.google.com/analytics), for [web analytics](/packages/analytics/web).
- [Polar](https://polar.sh), for [payments](/packages/payments).
- [Posthog](https://posthog.com), for [product analytics and error tracking](/packages/analytics).
- [Resend](https://resend.com), for [transactional emails](/packages/email).
