---
title: Payments
description: How next-ship handles payments and billing.
---

next-ship uses [Polar](https://polar.sh/) by default for payments and billing.

## In-App Purchases

You can use Polar anywhere in your server-side app code by importing the `polar` object:

```ts title="page.tsx {1,5}"
import { polar } from '@repo/payments';

// ...

await polar.products.list();
```

## Webhooks

next-ship has a pre-built webhook handler for payment events. Read more about [inbound payment webhooks](/packages/webhooks/inbound#payment-events).

## Configuration

Set the payment environment variables before enabling billing flows:

```bash
POLAR_ACCESS_TOKEN="polar_..."
POLAR_WEBHOOK_SECRET=""
POLAR_SERVER="sandbox"
```
